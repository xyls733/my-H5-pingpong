# 乒乓球AI动作分析助手

一个基于纯前端技术栈的乒乓球动作分析网页应用，利用MediaPipe Pose进行人体姿态识别，提供专业的动作评分和技术纠错建议。

## 功能特点

- 视频上传和摄像头实时拍摄支持
- 基于MediaPipe Pose的实时人体姿态检测
- 支持5种基础乒乓球动作识别（正手、反手、拉球、搓球、发球）
- 多维度动作评分系统（姿态规范性、动作连贯性、发力合理性、动作完整性）
- 详细的问题诊断和修正建议
- 动作轨迹可视化
- 关键帧标记和截图功能
- 分析报告导出（文本格式）
- 中英文双语支持
- 响应式设计，支持桌面端和移动端

## 技术架构

- **基础层**: HTML5、CSS3、原生JavaScript
- **AI视觉层**: MediaPipe Pose（人体2D姿态关键点实时检测）
- **图形渲染层**: Canvas API（骨架绘制、动作轨迹可视化）
- **数据处理层**: 原生JavaScript（动作数据提取、特征分析、规则匹配）

```
table-tennis-analyzer/
├── index.html              # 主HTML文件
```

### 2. 运行应用

由于这是一个纯前端应用，您可以直接在浏览器中打开`index.html`文件，或者使用本地服务器：

**使用Python简单HTTP服务器：**
```bash
python -m http.server 8000
```

**使用Node.js http-server：**
```bash
npx http-server
```

然后在浏览器中访问 `http://localhost:8000`

### 3. 使用方法

1. **上传视频**：点击"上传视频"按钮，选择本地的乒乓球运动视频文件（支持MP4、WebM、OGG格式）

2. **开启摄像头**：点击"开始摄像"按钮，允许浏览器访问摄像头进行实时分析

3. **分析动作**：点击"分析动作"按钮，系统将自动检测和分析视频中的乒乓球动作

4. **查看结果**：在右侧面板查看动作类型识别结果、综合评分、问题诊断和修正建议

5. **导出报告**：点击"导出报告"按钮，将分析结果导出为文本文件

## 核心功能说明

### 动作识别系统

应用内置了5种基础乒乓球动作的标准参数：
- 正手攻球
- 反手攻球
- 拉球
- 搓球
- 发球

每种动作都包含标准关节角度范围、动作时序特征和评分权重。

### 评分系统

评分系统基于4个维度进行综合评估：

1. **姿态规范性 (40%)**：评估关节角度是否符合标准动作要求
2. **动作连贯性 (30%)**：评估动作过程的流畅性和稳定性
3. **发力合理性 (20%)**：评估发力的时机和方式是否合理
4. **动作完整性 (10%)**：评估动作是否完整包含所有阶段

### 问题诊断

系统会自动识别动作中的技术问题，包括：
- 姿态规范性不足
- 动作连贯性差
- 发力不合理
- 关节角度异常
- 动作不完整

每个问题都会标注严重程度（严重/中等/轻微）和相关帧号。

### 可视化功能

- **骨骼点标注**：实时显示人体23个核心骨骼关键点
- **骨架绘制**：用线条连接骨骼点，形成人体骨架
- **轨迹追踪**：显示手腕运动轨迹，帮助分析挥拍路径

## 注意事项

1. 首次使用需要加载MediaPipe模型，可能需要几秒钟时间
2. 视频分析时间取决于视频长度和设备性能
3. 为获得最佳分析效果，建议：
   - 确保视频中人物全身可见
   - 光线充足，避免强烈阴影
   - 背景尽量简洁，避免复杂背景干扰
   - 动作完整，包含准备、引拍、击球、随挥等阶段

## 扩展和定制

### 添加新的动作类型

在`standard_actions.json`文件中添加新的动作定义：

```json
{
  "your_action": {
    "id": "your_action",
    "name": "动作名称",
    "nameEn": "Action Name",
    "jointAngles": {
      "elbow": { "min": 70, "max": 110, "optimal": 90 },
      ...
    },
    "sequence": [...],
    "scoringWeights": {...}
  }
}
```

### 调整评分权重

在`standard_actions.json`中修改各动作的`scoringWeights`参数来调整评分权重。

### 自定义问题诊断

在`app.js`中的`identifyIssues`方法中添加自定义的问题识别逻辑。
